About SENTINEL

AI-Driven Cyber Resilience for Critical National Infrastructure — Problem Statement 7

🎯 Overview

SENTINEL is a prototype system that demonstrates how behavioural anomaly detection combined with cross-source correlation can detect slow, low-and-slow Advanced Persistent Threats (APTs) that signature-based tools miss.

The system uses Isolation Forest — an unsupervised machine learning algorithm — to detect unusual patterns in network flow data and authentication logs independently. It then correlates weak signals across both sources using a sequential pattern match: a cluster of failed logins followed by unusual network activity on the same host within a follow-on window. This "compound risk" approach dramatically reduces false positives compared to either source alone.

🏗️ Architecture

1.

Data Generation

generate_data.py — Creates synthetic network flows and auth logs with embedded attack patterns

2.

Anomaly Detection

detect.py — Isolation Forest trains independently on each source, producing explainable anomaly scores

3.

Cross-Source Correlation

detect.py::correlate() — Matches auth clusters to network follow-ups on the same host

4.

MITRE ATT&CK Mapping

attack_mapping.py — Rule-based lookup table mapping anomalies to real ATT&CK technique IDs

5.

Web Dashboard

app.py — Flask server with login, API endpoints, and interactive UI

🔬 MITRE ATT&CK Techniques Detected
